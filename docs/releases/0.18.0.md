# Release Notes — pi-ptc-advanced 0.18.0

Date: 2026-05-13

## Summary

`0.18.0` closes Milestone 19 — Live Runtime Helper Hardening. The release tightens the live `code_execution` runtime surface so callable wrappers, normalization, and partial-error semantics behave consistently across the Phase 54–57 line, and ships a curated `ptc.list_helpers()` inventory as the counterpart to `ptc.list_callable_tools()`. The remaining live-audit edge cases are closed with focused regression coverage and explicit per-issue status.

## Added

- `ptc.list_helpers() -> list[dict[str, Any]]` — deterministic, JSON-safe inventory of `ptc.*` helpers (each entry has `name`, `signature`, and a one-line `summary`). Use it when Python needs to enumerate the helper surface; use `ptc.list_callable_tools()` to enumerate the callable Pi tool surface.
- Live-audit regression coverage in `test/live-audit-helpers.test.ts` for issues 1, 4, 5, 8, 9 (issue-numbered subtests for runner availability, helper-vs-callable distinction, positional-argument behavior, `details.ptcValue` preservation, and shell-quoted command reporting).

## Changed

- `find`, `glob`, and `ls` Python wrappers accept at most one positional argument and raise a wrapper-named `TypeError` on excess positionals — matching the existing behavior of `read` and `grep` (Phase 55 + Phase 57).
- `ptc.run_tests(pattern)` `metrics.command` is shell-quoted via `shlex.join`, and `metrics.runner_available` is a bool surfaced in the report (Phase 54, pinned in Phase 57).
- `ptc.read_many(...)` accepts opt-in `on_error='collect'` and returns a typed partial envelope (`kind="read_many_partial"`) instead of raising on the first failure (Phase 56).
- `ptc.batch_tool(..., on_error='collect')` classifies tool-level normalized error payloads as failed entries (`ok: false, error: summary`) while preserving the raw payload under `value` (Phase 56).
- Path normalization invariant documented for direct callable wrappers, `ptc.read_*` helpers, and `ptc.batch_tool` read/grep payloads (Phase 56).
- README and the generated `code_execution` tool description distinguish `ptc.list_callable_tools()` (callable Pi tools) from `ptc.list_helpers()` (`ptc.*` helpers). The `details.ptcValue` callable-boundary callout remains in README to preserve the prompt-description invariant.
- Package metadata and release-package verification target `pi-ptc-advanced@0.18.0`.
- `docs/issues/2026-05-13-code-execution-helper-edge-cases.md` records explicit per-issue resolution status for all ten live-audit items.
