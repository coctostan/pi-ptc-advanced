<objective>
## Goal
Make the personal fork straightforward to operate and maintain by adding repo-local maintenance entrypoints, documenting the day-to-day workflow, and capturing an explicit manual upgrade boundary that does not depend on `.paul/**` history.

## Purpose
Phase 19 restored the personal analysis profile and clarified the callable-tool surface. Phase 20 should turn that into a durable local-maintenance workflow so future sessions do not have to rediscover how to launch, verify, and update the fork.

## Output
- A repo-local maintenance verification script with stable package-script entrypoints
- A maintainer-facing runbook for daily use, verification, and manual upgrade/sync steps
- README and launcher guidance that point to the same personal-fork workflow
</objective>
