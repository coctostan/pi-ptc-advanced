<objective>
## Goal
Make the fork present a coherent 0.8.0 release/package surface by aligning version markers, filling the missing package metadata and license gap, and adding a repeatable package-verification entrypoint.

## Purpose
Milestone 8 made the fork operational for personal maintenance, but the repo still has a release-facing mismatch: the milestone roadmap is now at 0.8.0 while the package surfaces still say 0.1.1, there is no top-level license file, and there is no stable repo-local command that proves what would actually ship in a package tarball. Phase 22 should fix that release-packaging baseline before Phase 23 writes release notes and Phase 24 adds CI/release verification automation.

## Output
- Version-aligned project/package metadata for the 0.8.0 milestone
- A complete package surface with explicit repository metadata and a checked-in MIT license file
- A repo-local release-package verification command documented in the normal maintainer docs
</objective>
