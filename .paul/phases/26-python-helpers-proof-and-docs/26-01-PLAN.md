<objective>
## Goal
Add bounded Python-side handle helper ergonomics, proof, and documentation for the Phase 25 response/file handle contract so `code_execution` users can follow existing `responseId` and `filePath` workflows without ad hoc JSON walking.

## Purpose
Phase 25 made the supported response/file handle contract explicit inside `pi-ptc-next`; this phase should make that contract practical from Python while preserving the current runtime shape guarantees. The result should help models and maintainers use existing `pi-web-tools` follow-up flows cleanly without widening scope into graph handles or changing `normalizeToolResult()` behavior.

## Output
- Typed Python helper surface for bounded response/file handle extraction
- Focused execution proof covering supported follow-up workflows
- README/model-guidance updates that document the live helper API and its limits
</objective>
