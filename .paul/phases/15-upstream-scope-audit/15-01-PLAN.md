<objective>
## Goal
Produce an explicit upstream scope audit for `pi-ptc-next` that identifies which current changes are PR-worthy versus local-only, recommends the reviewable PR split, and records the cleanup/hardening inputs needed for the next phases.

## Purpose
This phase converts the recent implementation and diagnosis work into a reviewer-oriented scope boundary. That keeps the project aligned with its core value while preventing upstream submission from being diluted by personal tooling, PALS artifacts, or repo-specific compatibility workarounds.

## Output
- `.paul/phases/15-upstream-scope-audit/15-01-UPSTREAM-SCOPE.md` containing:
  - current change inventory
  - keep / exclude classification with rationale
  - recommended PR package split and ordering
  - explicit cleanup and hardening checklist for Phases 16-18
</objective>
