<objective>
## Goal
Align the extension runtime surface with the latest Mario-scope Pi line (`@mariozechner/*@0.73.1`) and the runtime compatibility findings from Phase 45 without changing `code_execution` user behavior.

## Purpose
`pi-ptc-advanced` should compile against the same active Pi APIs that users run locally, keep hashline-native tool execution observable, and make Pi `sourceInfo` handling deliberate instead of accidental.

## Output
- Package metadata and lockfile aligned to the latest Mario-scope Pi packages (`@mariozechner/pi-coding-agent@0.73.1` and `@mariozechner/pi-tui@0.73.1`).
- Existing `@mariozechner/*` source imports preserved so the extension continues to build/load for the user's current Mario-scope Pi setup.
- Typed `pi.on("context", ...)` registration without the obsolete cast.
- Tool registry hashline bridge warning when no executors are visible at session start.
- A documented `sourceInfo` strategy that preserves PTC's internal source taxonomy while carrying Pi provenance where useful.
</objective>
