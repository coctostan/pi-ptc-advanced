<objective>
## Goal
Audit the current `code_execution` Python-source visibility path for running, completed-success, and failed executions, then produce a concrete implementation map for Milestone 21's payload and Pi TUI work.

## Purpose
The user reports that Pi TUI currently shows only the `code_execution` result, not the Python code the agent wrote. Project history shows Phase 49 already added completed-result source visibility via `details.userCode` and Pi's expanded tool-result affordance, so Phase 62 must establish what actually exists now, what is hidden/insufficient, and which gap future phases should fix: payload contract, streaming/error details, TUI rendering, or collapse/expand UX.

## Output
A single audit artifact at `.paul/phases/62-current-behavior-audit/62-01-SOURCE-VISIBILITY-AUDIT.md` that records:
- current source visibility behavior for running, success, and failure states;
- code/test evidence paths and relevant symbols;
- gap classification against the user's desired behavior (collapsed-by-default first-line preview, expanded full source, all states);
- recommended Phase 63 and Phase 64 implementation boundaries.
</objective>
