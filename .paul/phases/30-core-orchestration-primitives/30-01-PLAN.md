<objective>
## Goal
Add bounded high-level orchestration helpers to the Python runtime so model-written code can execute small batches of nested tool calls and try multiple candidate calls without repeating low-level `_rpc_call(...)` plumbing in user code.

## Purpose
Milestone 12 Phase 30 should make common multi-tool orchestration flows shorter and safer while preserving the narrow helper philosophy established in Milestones 10 and 11. This phase should land only the first core orchestration slice: `ptc.batch_tool(...)` and `ptc.first_success(...)`. It must stay generic, provider-agnostic, and small, without drifting into reducers, output-budget helpers, `parallel_map(...)`, or user-facing docs.

## Output
- `ptc.batch_tool(...)` in the Python runtime
- `ptc.first_success(...)` in the Python runtime
- A bounded call-spec validation path shared by both helpers
- Focused execution-level proof for the new helpers in a dedicated test file
</objective>
