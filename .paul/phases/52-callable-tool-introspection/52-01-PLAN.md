<objective>
## Goal
Extend callable-tool metadata surfaced inside Python so agents can inspect concise prompt metadata at runtime via `ptc.help(tool_name)` without inflating the `code_execution` tool-description block.

## Purpose
Phase 47 made `promptSnippet` / `promptGuidelines` visible through Pi's native tool registration path. Phase 52 brings that same optional metadata inward to `code_execution` callable-tool introspection so Python scripts can ask what an optional callable tool is for before choosing it, while preserving existing `ptc.list_callable_tools()` / `ptc.get_tool_schema()` behavior.

## Output
- Callable-tool metadata includes optional `promptSnippet` and `promptGuidelines` when Pi/custom tools provide them.
- Runtime helper `ptc.help(tool_name)` returns a bounded JSON-safe metadata object, likely with name, pythonName, description, promptSnippet, promptGuidelines, parameters, source, and read-only status.
- README and generated tool-description guidance document `ptc.help(...)` as an on-demand introspection helper, not as a reason to call introspection unconditionally.
</objective>
