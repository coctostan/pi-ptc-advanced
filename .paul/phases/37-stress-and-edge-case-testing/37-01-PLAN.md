    Create `test/live-audit-stress.test.ts` using the same CodeExecutor + stub-tool harness from Phase 36. Reuse the `createAuditExecutor` pattern (spawn real python3, stub tool responses).

    The stub tools should support:
    - `read` — returns configurable content (large files for stress)
    - `slow_tool` — adds a configurable delay before returning (for concurrency testing)
    - `failing_tool` — always throws an error (for error chain testing)

    **Concurrency stress tests (~5 tests):**
    1. `ptc.batch_tool()` with 10 parallel read calls — assert all 10 return, no drops
    2. `ptc.batch_tool()` with max_concurrency=2 and 6 calls with delays — assert max active calls ≤ 2
    3. `ptc.gather_limit()` with 8 coroutines, limit=3 — assert all 8 complete, concurrency bounded
    4. `ptc.batch_tool()` where 2 of 5 calls fail — assert the successful ones still return
    5. `ptc.first_success()` with 3 failing calls then 1 success — assert fallback chain works

    **Large file / output budget stress (~4 tests):**
    6. `read()` a 1000-line stub file — assert full content returned
    7. `ptc.read_many()` with 5 large files (500+ lines each) — assert all return
    8. `ptc.fit_output()` with deeply nested structure (5 levels, 200 items) — assert truncation works cleanly
    9. `ptc.fit_output()` with max_chars near the session limit — assert bounded output, no crash

    **Error and edge case tests (~5 tests):**
    10. `ptc.batch_tool()` with an invalid tool name — assert clear error
    11. `ptc.get_tool_schema("nonexistent_tool")` — assert returns empty/error gracefully
    12. `ptc.expect_kind()` with wrong kind — assert ValueError raised
    13. `ptc.reduce_tool()` with all-failing calls — assert error propagation
    14. Empty inputs: `ptc.batch_tool([])`, `ptc.read_many([])` — assert graceful empty results

    Each test should use the same executor pattern from Phase 36. Track concurrency via the stats object for concurrency tests.

    Keep the file compact (~200-300 lines). Focus on stress conditions, not re-testing basic functionality.
