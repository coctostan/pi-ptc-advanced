<objective>
## Goal
Add bounded Python-side tool introspection helpers so model-written code can inspect the current callable tool surface and fetch a tool's parameter schema without scraping the `code_execution` description or reaching into internal runtime structures.

## Purpose
Milestone 11 Phase 28 should make the live callable runtime surface programmatically inspectable from Python while preserving the narrow helper philosophy established in Phase 27. The work should stay focused on session-backed introspection for callable tools only, without drifting into broader docs work, execution-level demos, or a general schema/registry framework.

## Output
- `ptc.list_callable_tools()` in the Python runtime
- `ptc.get_tool_schema(name)` in the Python runtime
- Generated session-scoped callable-tool metadata derived from the real callable runtime surface
- Focused contract/unit coverage for wrapper metadata emission and introspection plumbing
</objective>

## AC-1: Callable tool listings are available inside Python from the real session surface
```gherkin
Given a `code_execution` session with a specific callable tool set and any Python helper aliases applied
When Python calls `ptc.list_callable_tools()`
Then it receives a deterministic list derived from that live callable runtime surface, including enough metadata to identify each tool by canonical name and Python helper name without exposing runtime executors
```

## AC-2: Python can fetch a callable tool's parameter schema with clear failures
```gherkin
Given a callable tool that is present in the current session
When Python calls `ptc.get_tool_schema(name)` using its helper name or canonical tool name
Then it receives the JSON-schema-like parameter definition for that callable tool
And when the name is blank or unknown, execution fails with a clear bounded error
```

```gherkin
Given the planned Milestone 11 phase split
When Phase 28 lands the introspection helpers
Then the work is limited to runtime/helper metadata plumbing plus focused contract coverage
And README/tool-description updates, safe usage examples, and execution-level proof remain out of scope for this phase
```
